# Preferencias de colaboración

Cómo quiere el usuario que trabaje. Leer al iniciar y respetar.
Agregar cuando el usuario corrige un enfoque o confirma uno no obvio.

## Estilo de respuesta
- Respuestas cortas y directas en español.
- Sin emojis salvo pedido explícito.
- Explicar cambios brevemente tras hacerlos, no antes.

## Flujo de trabajo
- Commits: descriptivos, tipo `feat(scope): ...` / `fix(scope): ...`.
- Push obligatorio después de cada cambio relevante (no tiene entorno local).
- Aplicar migraciones a prod solo si el usuario lo pide explícitamente.
- Preferir editar archivos existentes antes que crear nuevos.

## Unidades
- Todas las medidas del juego en metros. Convención: 3 ft ≈ 1 m, 1 casilla = 2 m.
